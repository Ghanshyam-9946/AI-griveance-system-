# AI-Based Smart Grievance Redressal System
## Project Presentation Report

---

## 1. Project Overview
The **AI-Based Grievance System** is a next-generation platform designed to streamline the reporting and resolution of civic issues (like potholes, water leaks, garbage dumps, etc.). By leveraging state-of-the-art Artificial Intelligence (AI) and Machine Learning (ML) models, the system completely automates complaint categorization, priority assignment, spam detection, and fake-image detection, making the workflow incredibly efficient for government departments.

---

## 2. Comprehensive Tech Stack
The project follows a modern **3-Tier Microservices Architecture**, ensuring scalability and high performance.

### 🌐 Frontend (User & Admin Interface)
*   **Core:** React.js powered by Vite (for extremely fast builds and hot-reloading).
*   **Styling:** Tailwind CSS (Utility-first framework for responsive UI).
*   **Animations:** Framer Motion (for smooth, dynamic micro-interactions).
*   **Maps & Geolocation:** React Leaflet & OpenStreetMap (for precise location tagging of grievances).
*   **Localization:** `react-i18next` (Supports multiple regional languages for wider accessibility).
*   **HTTP Client:** Axios.

### ⚙️ Backend (API Server)
*   **Core:** Node.js with Express.js framework.
*   **Database:** MongoDB (using Mongoose ORM for schema validation).
*   **Authentication:** JWT (JSON Web Tokens) & `bcryptjs` for secure department login.
*   **User Verification:** Twilio SMS API (for Aadhaar-linked OTP verification).
*   **File Handling:** Multer (for handling image uploads from users).

### 🧠 AI Service (Machine Learning Engine)
*   **Core:** Python with FastAPI (High-performance async API framework).
*   **ML Library:** PyTorch & Hugging Face `transformers`.
*   **Models Deployed:**
    1.  **BART (`facebook/bart-large-mnli`)**: Used for *Zero-Shot Text Classification*. It reads the grievance description and automatically assigns the correct **Category** (e.g., Road, Water) and **Priority** (High, Medium, Low).
    2.  **BLIP (`Salesforce/blip-image-captioning-base`)**: An advanced Vision-Language model that "looks" at the uploaded image and generates a text description to verify the user's claim.
    3.  **CLIP (`openai/clip-vit-base-patch32`)**: Used for *Semantic Image Comparison*. It detects duplicate complaints by comparing the visual similarity of a newly uploaded image with existing database images (Spam Prevention).
    4.  **AI Image Detector (`umm-maybe/AI-image-detector`)**: A specialized model that scans images to check if they were generated by AI (Midjourney/DALL-E) to prevent fake or malicious grievance submissions.

## 3. System Workflow (Step-by-Step Flow)

### Step 1: Secure User Onboarding
*   The user accesses the platform and logs in using their **Aadhaar Number** and Phone Number.
*   The backend triggers the **Twilio API** to send a secure OTP for verification.

### Step 2: Grievance Submission
*   The user fills out the grievance form, drops a pin on the **Interactive Map** to log exact GPS coordinates, and uploads a live photo of the issue.
*   The frontend sends this payload to the Node.js Backend.

### Step 3: AI Processing & Verification (The Magic)
*   Before saving to the database, the backend forwards the image to the **FastAPI AI Service**.
*   **Fake Detection:** The AI checks if the image is synthetically generated. If it's a fake AI image, the complaint is instantly rejected.
*   **Duplicate Check:** The CLIP model converts the image into vector embeddings and calculates the "Cosine Similarity" with recent complaints within a 100-meter radius. If a match is found, it is flagged as a duplicate.
*   **Context Generation:** The BLIP model captions the image (e.g., "A deep pothole on a tar road").
*   **Smart Routing:** The BART model reads this context and assigns a category ("Road Issues") and calculates the priority ("High").

### Step 4: Storage & Gamification
*   If the grievance passes all AI checks, it is stored in **MongoDB**.
*   The user is instantly rewarded with **Civic Tokens** (Gamification) to encourage active participation in maintaining the city.

### Step 5: Department Action (Admin Panel)
*   Department officials log in using secure credentials (JWT).
*   They see a filtered dashboard of grievances relevant *only* to their department, sorted automatically by AI-assigned priority.
*   The official can update the status to "In Progress" or "Resolved" and upload a "Resolution Proof" image.

## 4. Key Unique Selling Propositions (USPs)
1.  **Zero Manual Triage:** The AI completely replaces human operators for categorizing and prioritizing complaints.
2.  **Anti-Spam & Anti-Fake:** First-of-its-kind integration of AI image detection in a civic platform.
3.  **Gamified Citizen Engagement:** Token reward system incentivizes users to report real issues.
4.  **Multilingual:** Breaks the language barrier by allowing users to interact in their native language.
